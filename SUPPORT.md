# Support and version policy

HandUMI is a research preview. The default branch and latest tagged research
preview receive best-effort software fixes; older snapshots and unlisted
hardware/runtime combinations may not receive backports.

Use GitHub issues for reproducible software defects and feature proposals. Do
not post vulnerabilities, participant data, private addresses, device
identifiers, signing material, or unsafe robot incidents publicly; follow
SECURITY.md instead. Include the source commit, Python/runtime versions,
selected dependency groups, operating system, redacted configuration, and a
minimal synthetic reproducer.

A simulator or kinematic model being present does not imply hardware support.
Supported hardware, Quest/Horizon OS, Unity/Meta XR/OpenXR, Python, and optional
integration versions must be listed in the release notes with evidence. In the
absence of that matrix, treat the combination as unvalidated.

The project provides no medical, ergonomic, anatomical-accuracy, timing,
balance, contact, support, or safety qualification. Real-robot help never
replaces the manufacturer's procedures, laboratory approval, independent
safety controls, or trained on-site supervision.
